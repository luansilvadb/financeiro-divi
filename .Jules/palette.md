# Palette's Journal - UX & Accessibility Learnings

## 2025-05-15 - [Password Visibility Consistency]
**Learning:** Even if primary authentication forms (Login/Register) include password visibility toggles, administrative or secondary forms (like adding a new house member) often overlook them, leading to a fragmented user experience and potential frustration when entering complex passwords.
**Action:** Always verify that every password input field in the application includes a visibility toggle, ensuring consistency across all user flows. Use the `Eye`/`EyeOff` icon pattern with a reactive state for uniform behavior.

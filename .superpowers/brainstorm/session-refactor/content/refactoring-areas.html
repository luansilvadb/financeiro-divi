Refatoração e Limpeza: Onde começar?

Identifiquei três áreas principais que podem ser simplificadas para melhorar a manutenção e reduzir a complexidade.

A

Quebra de Componentes "Gigantes"

Extrair sub-componentes de NovoLancamentoWizard.vue e DashboardSaldos.vue para torná-los mais focados e testáveis.

B

Extração de Composables

Mover a lógica de persistência de rascunhos (localStorage) e gerenciamento de passos do Wizard para composables reutilizáveis.

C

Padronização de Primitivos de UI

Criar componentes base para inputs, botões e cards, eliminando a repetição massiva de classes Tailwind no código.

Minha Recomendação

Eu recomendo começar pela Extração de Composables (B). Isso vai limpar instantaneamente os componentes principais de lógica "não-visual", facilitando muito a quebra subsequente dos componentes (A).

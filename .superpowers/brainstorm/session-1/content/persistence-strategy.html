Estratégia Local-First (Mocked Adapters)

Como simularemos a persistência antes do Supabase.

A

LocalStorage Adapter

Persistência simples no navegador. Ideal para testar o fluxo de ponta a ponta sem internet.

B

In-Memory Adapter

Dados resetam ao atualizar a página. Foco total em testes de unidade e velocidade de desenvolvimento.

Proposta de Schema Relacional

Mesmo local-first, vamos modelar como se fosse SQL para facilitar a migração futura:

  • Casas: id, nome, membros_ids.
  • Membros: id, nome, email.
  • Transações: id, total, fonte_id, pagador_id, status.
  • Divisões: transacao_id, beneficiario_id, valor.
